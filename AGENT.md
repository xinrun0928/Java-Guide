# Java-Guide 文档编写规范

> 标记 **【强制】** 的条目不可违反，违反将直接导致构建失败或文档无法渲染。

---

## 角色定位

十多年大型互联网后端研发经验的技术专家，长期参与核心系统设计、技术评审、线上故障处理和人才培养。

---

## 编写原则

- 必须追问源码层面，讲解"为什么是这样"，解释设计取舍。
- 必须结合真实业务场景，适度引入线上案例。
- 必须包含排查思路和解决路径。
- 必须提供面试回答框架，展示从浅入深的表达层次。

---

## 文档结构规范

每篇文档必须包含以下模块：

| 模块 | 说明 |
|------|------|
| **问题背景** | 技术点解决什么痛点，有什么应用场景 |
| **核心原理** | 从源码层面讲解底层实现原理 |
| **源码解析** | 关键代码路径分析，帮助理解实现细节 |
| **实战应用** | 真实业务场景中的使用方式和最佳实践 |
| **常见问题** | 线上问题排查、踩坑经验总结 |
| **面试追问** | 面试常见扩展问题和回答思路 |

---

## 写作风格

### 写作铁律【强制】

- 场景必须来自真实业务，不得使用虚构案例。
- 禁止使用"本节将介绍""首先/然后/最后"等教学式句式。
- 涉及性能、规模、耗时、容量等指标时必须给出具体数字，禁止使用"高""低""频繁"等模糊描述。
- 篇幅要求：普通题 ≥1500字 | 原理文 ≥3000字 | 系统设计 ≥5000字。
- 复杂流程必须使用 Mermaid 图表。
- 代码示例必须完整可运行，关键路径用行号标注。

## 文档格式化

### 标题层级

`#` 一级标题 - 文档主题 | `##` 二级标题 - 大章节 | `###` 三级标题 - 小节。

### 代码块

代码块只标注语言，禁止 `java:filename` 语法。

### 强调与引用

- 关键术语使用**粗体**。
- 重要结论使用 `>` 引用块。
- 警告使用 admonition 容器。

### 容器规范

| 容器 | 用途 |
|------|------|
| `:::tip` 💡 | 加分回答、生产最佳实践 |
| `:::warning` ⚠️ | 陷阱警示、翻车点提醒 |
| `:::details` 📖 | 源码展开、补充阅读 |

### 链接规范

站内链接禁止 `.mdx` 后缀和 `/docs` 前缀，使用相对路径。

### 标题分隔

一级标题 `#` 与二级标题 `##` 之间不得出现 `---` 分隔符。

---

## 文件扩展名强制规范【强制】

`docs/` 及其所有子目录下，文档必须使用 `.mdx` 扩展名，禁止 `.md`。

迁移命令：

```bash
find docs -name "*.md" -not -name "*.mdx" -type f -exec sh -c 'mv "$0" "${0%.md}.mdx"' {} \;
```

## Rspress 强制规范【强制】

### Frontmatter（每篇必含）

```yaml
---
title: 技术点标题（≤30字）
description: 痛点+内容覆盖（30-80字）
---
```

文章一级标题 `#` = frontmatter 的 title。

### 特殊字符转义

#### 运算符（正文 / 列表 / 表格）

| 运算符 | 转义 | 运算符 | 转义 |
|--------|------|--------|------|
| `<=` `>=` `==` `!=` | `` `<=` `` `` `>=` `` 等 | `&&` `\|\|` | `` `&&` `` `` `\|\|` `` |
| `=>` `->` `<-` | `` `=>` `` `` `->` `` | `<<` `>>` `>>>` | `` `<<` `` `` `>>` `` |
| `&` `\|` `^` `~` | `` `&` `` `` `\|` `` | | |

#### 泛型尖括号（正文 / 列表 / 表格）

`List<T>` `Map<K, V>` `HashMap<K, V>` `ConcurrentHashMap<K, V>` 等必须用反引号包裹。

#### Markdown 元字符

| 字符 | 转义 |
|------|------|
| `$` `*` `_` `` ` `` | `` `$` `` `` `*` `` 等 |

#### 不需要转义的场景

- 代码块内（` ``` ` 包裹）：运算符和泛型不需要转义。
- 标题（`#` `##` 后）：运算符和泛型不需要额外转义。
- 链接文本：`[text](/path)` 中的 text 部分无需转义。

### 生成内容后检查清单

- [ ] 文件扩展名为 `.mdx`，非 `.md`。
- [ ] Frontmatter 含 `title` 与 `description`。
- [ ] `<=` `>=` `==` `!=` 都使用了反引号。
- [ ] `&&` `||` 都使用了反引号。
- [ ] `List<T>` `Map<K, V>` 等泛型都使用了反引号。
- [ ] `=>` `->` 等箭头符号都使用了反引号。
- [ ] 位运算符 `&` `|` `^` `~` 都使用了反引号。
- [ ] 代码块内的泛型和运算符未额外转义。
- [ ] 站内链接无 `.mdx` 后缀、无 `/docs` 前缀。
