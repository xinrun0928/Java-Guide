# Kafka 存储机制与日志分段

> 上一节 [Kafka 消息丢失与重复消费](/fw/mq/kafka/message-loss) 讨论了可靠性边界，Kafka 能高吞吐写入，底层存储机制功不可没。

## 存储结构

Kafka 的消息存储在 `data/log/` 目录下：

```
/var/lib/kafka/data/
├── topic-order-events/
│   ├── partition-0/
│   │   ├── 00000000000000000000.index
│   │   ├── 00000000000000000000.timeindex
│   │   ├── 00000000000000000000.log
│   │   ├── 00000000000000000001.snapshot
│   │   └── leader-epoch-checkpoint
│   ├── partition-1/
│   │   └── ...
│   └── partition-2/
│       └── ...
```

每个 Partition 包含多个 Segment（分段），每个 Segment 由 `.log`、`.index`、`.timeindex` 组成。

## 日志分段（Log Segment）

日志分段是 Kafka 存储的核心机制：

| 文件 | 作用 |
|------|------|
| `.log` | 存储消息内容 |
| `.index` | 消息偏移量索引 |
| `.timeindex` | 时间戳索引 |

### 分段规则

```properties
# 分段大小（默认 1GB）
log.segment.bytes=1073741824

# 分段文件滚动时间（默认 7 天）
log.retention.check.interval.ms=300000
```

当满足以下任一条件时，创建新 Segment：
- 当前 Segment 文件大小达到 `log.segment.bytes`
- 消息时间戳超过当前时间 `log.roll.jms.hours`

## 顺序写入原理

Kafka 利用磁盘顺序写入实现高吞吐：

```java
// 写操作：追加到末尾
FileOutputStream.write(bytes);  // 顺序写入

// 读操作：利用索引快速定位
index.seek(targetOffset);       // 索引定位
```

**顺序写入速度**：机械硬盘可达 200MB/s，SSD 可达 1GB/s。

## 索引机制

### 偏移量索引

`.index` 文件存储消息偏移量到物理位置的映射：

```bash
# 查看 index 内容
kafka-dump-log.sh --print-data-log --files 00000000000000000000.index

# 输出示例
offset: 5 position: 1024
offset: 12 position: 2048
offset: 20 position: 3072
```

### 时间索引

`.timeindex` 存储时间戳到偏移量的映射，支持按时间查询：

```bash
# 按时间范围消费
consumer.seekToTime(Duration.ofHours(1));
```

## 高效写入的关键

| 优化点 | 说明 |
|--------|------|
| 顺序写入 | 避免磁盘寻道 |
| 批量发送 | 减少网络往返 |
| 零拷贝 | 直接从 PageCache 发送到网卡 |
| PageCache | 操作系统内存缓存热点数据 |
