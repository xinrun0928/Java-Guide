# 消息队列

> 异步处理、削峰填谷、系统解耦 —— 消息队列是分布式架构的粘合剂。

## 核心问题

使用消息队列前，先问自己三个问题：

| 问题 | 答案方向 |
|------|----------|
| 能否接受最终一致性？ | 是 → MQ；否 → 分布式事务 |
| 消息丢失能接受吗？ | 不能 → 持久化 + 确认机制 |
| 需要消息顺序吗？ | 是 → 单分区顺序消费 |
