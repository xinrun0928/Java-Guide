# 延迟队列与延迟插件

> 上一节 [死信队列](/fw/mq/rabbitmq/dead-letter) 提到用 DLX 实现延迟消息，本节详细介绍延迟队列的各种实现方式。

## 延迟消息的三种实现

| 方式 | 延迟精度 | 消息堆积 | 复杂度 |
|------|----------|----------|--------|
| TTL + DLX | 秒级 | 支持 | 低 |
| 死信插件 | 毫秒级 | 不支持 | 中 |
| 延迟交换机插件 | 毫秒级 | 支持 | 低 |

## 方式一：TTL + 死信队列

利用消息或队列的 TTL，过期后进入死信队列：

```java
// 1. 声明延迟队列（设置 TTL）
Map<String, Object> args = new HashMap<>();
args.put("x-message-ttl", 60000);  // 60 秒
args.put("x-dead-letter-exchange", "order-exchange");
args.put("x-dead-letter-routing-key", "order.process");

channel.queueDeclare("order-delay-queue", true, false, false, args);

// 2. 正常交换机
channel.exchangeDeclare("order-exchange", BuiltinExchangeType.DIRECT, true);
channel.queueDeclare("order-process-queue", true, false, false, null);
channel.queueBind("order-process-queue", "order-exchange", "order.process");

// 3. 发送延迟消息
channel.basicPublish("", "order-delay-queue", properties, orderMessage);
```

**问题**：消息按队列顺序到期，无法实现"插队"。

## 方式二：死信插件（rabbitmq_delayed_message_exchange）

安装插件后，支持任意时间延迟：

```bash
# 安装插件
rabbitmq-plugins enable rabbitmq_delayed_message_exchange

# 查看插件
rabbitmq-plugins list
```

### 配置

```java
// 声明延迟交换机
Map<String, Object> args = new HashMap<>();
args.put("x-delayed-type", "direct");

channel.exchangeDeclare("order-delay-exchange",
    "x-delayed-message",  // 交换机类型
    true,                 // durable
    false,                // autoDelete
    args);

// 发送延迟消息（x-delay 头指定延迟毫秒数）
AMQP.BasicProperties properties = new AMQP.BasicProperties.Builder()
    .deliveryMode(2)
    .headers(Map.of("x-delay", 30000))  // 30 秒后投递
    .build();

channel.basicPublish("order-delay-exchange", "order.created", properties, body);
```

## 方式三：多队列分级延迟

将延迟时间分级，用多个队列：

```java
// 5 秒队列
channel.queueDeclare("delay-5s", true, false, false,
    Map.of("x-message-ttl", 5000,
           "x-dead-letter-exchange", "order-exchange"));

// 30 秒队列
channel.queueDeclare("delay-30s", true, false, false,
    Map.of("x-message-ttl", 30000,
           "x-dead-letter-exchange", "order-exchange"));

// 5 分钟队列
channel.queueDeclare("delay-5m", true, false, false,
    Map.of("x-message-ttl", 300000,
           "x-dead-letter-exchange", "order-exchange"));
```

## 三种方式对比

| 特性 | TTL+DLX | 延迟插件 | 分级队列 |
|------|---------|----------|----------|
| 延迟精度 | 秒级 | 毫秒级 | 秒级 |
| 消息堆积 | 支持 | 支持 | 支持 |
| 插队能力 | ❌ | ✅ | ❌ |
| 任意延迟 | ❌ | ✅ | ❌ |
| 性能 | 高 | 中 | 高 |

## 应用场景选择

| 场景 | 推荐方式 |
|------|----------|
| 订单超时取消（分钟级） | TTL+DLX |
| 精确延迟通知（秒/毫秒） | 延迟插件 |
| 固定延迟任务 | 分级队列 |

## 面试回答框架

**问题**：RabbitMQ 如何实现延迟消息？

**回答**：
1. **TTL + 死信队列**：消息设置 TTL，过期后进入死信队列
2. **延迟交换机插件**：安装 `rabbitmq_delayed_message_exchange`，支持任意时间延迟
3. **分级队列**：多个不同 TTL 的队列，适合固定延迟
4. 注意：TTL+DLX 无法保证插队，消息按进入顺序到期
