# RabbitMQ 权限管理与安全

> 上一节 [RabbitMQ 集群与镜像队列](/fw/mq/rabbitmq/cluster) 讲解高可用集群，本节讲解如何安全管理 RabbitMQ。

## 用户管理

### 创建用户

```bash
# 创建用户
rabbitmqctl add_user admin strong_password

# 设置管理员标签
rabbitmqctl set_user_tags admin administrator

# 设置密码
rabbitmqctl change_password admin new_password
```

### 用户标签

| 标签 | 权限 |
|------|------|
| (无标签) | 普通用户 |
| `monitoring` | 监控查看 |
| `management` | HTTP API 管理 |
| `policymaker` | 策略管理 |
| `administrator` | 完全控制 |

### 用户权限级别

| 级别 | 说明 |
|------|------|
| `none` | 无法访问 |
| `configure` | 管理 Exchange、Queue |
| `write` | 发送消息 |
| `read` | 消费消息 |

## 权限控制

### 设置权限

```bash
# 设置 read/write/configure 权限
rabbitmqctl set_permissions -p / admin ".*" ".*" ".*"

# 只读权限
rabbitmqctl set_permissions -p / readonly_user "^$" "^$" ".*"
```

### 基于 vhost 的隔离

```bash
# 创建虚拟主机
rabbitmqctl add_vhost /order
rabbitmqctl add_vhost /payment

# 用户绑定 vhost
rabbitmqctl set_permissions -p /order order_user "order.*" "order.*" "order.*"
rabbitmqctl set_permissions -p /payment payment_user "payment.*" "payment.*" "payment.*"
```

### 代码配置

```java
ConnectionFactory factory = new ConnectionFactory();
factory.setHost("localhost");
factory.setPort(5672);
factory.setUsername("order_user");
factory.setVirtualHost("/order");
```

## 生产环境安全配置

### 1. 端口安全

```properties
# rabbitmq.conf
listeners.tcp.default = 5672
management.tcp.port = 15672

# 禁用 guest 用户远程访问
loopback_users.guest = true
```

### 2. 连接限制

```properties
# 最大连接数
max_keep_connections = 100

# 单用户最大连接数
max_connections_per_user = 50
```

### 3. 资源限制

```bash
# 设置用户资源限制
rabbitmqctl set_limits order_user max_connections 100
rabbitmqctl set_limits order_user max_queues 50
```

## TLS/SSL 配置

```properties
# rabbitmq.conf
listeners.ssl.default = 5671

ssl.options.verify = verify_peer
ssl.options.fail_if_no_peer_cert = true
ssl.cacertfile = /path/to/ca_certificate.pem
ssl.certfile = /path/to/server_certificate.pem
ssl.keyfile = /path/to/server_key.pem
```

### 生产者 TLS 配置

```java
ConnectionFactory factory = new ConnectionFactory();
factory.setHost("localhost");
factory.setPort(5671);
factory.useSslProtocol();

// 客户端证书（如果需要双向认证）
```

## 审计日志

```bash
# 开启审计
rabbitmqctl set_plugin_attr_enabled rabbitmq_management_enhanced true

# 查看审计日志
rabbitmqctl log_category set_connection true
```

## 常见安全问题

| 问题 | 风险 | 解决 |
|------|------|------|
| Guest 默认用户 | 未授权访问 | 删除或设置强密码 |
| 无连接限制 | 资源耗尽 | 设置连接数限制 |
| 无 vhost 隔离 | 跨业务干扰 | 按业务划分 vhost |
| 明文传输 | 数据泄露 | 启用 TLS |

## 最佳实践清单

- [ ] 删除 guest 默认用户
- [ ] 使用强密码或 LDAP 认证
- [ ] 按业务划分 vhost
- [ ] 最小权限原则分配用户权限
- [ ] 启用 TLS 加密传输
- [ ] 设置连接数限制
- [ ] 开启审计日志
- [ ] 定期轮换密码

## 面试回答框架

**问题**：RabbitMQ 如何进行权限管理？

**回答**：
1. 用户管理：创建用户，设置标签（administrator/management）
2. 虚拟主机（vhost）：按业务隔离
3. 资源权限：read/write/configure 三种权限
4. 连接限制：防止资源耗尽
5. TLS 加密：生产环境建议启用
